# 提示工程

## 1. 基础提示技巧

### 1.1 Zero-Shot Prompting
直接给任务描述，不提供示例：
```
请将以下文本翻译成英文：今天天气很好
```

### 1.2 Few-Shot Prompting
提供少量示例，引导模型理解任务：
```
请将以下文本翻译成英文：
猫 → cat
狗 → dog
鸟 → bird
鱼 → fish
今天天气很好 →
```

### 1.3 Chain-of-Thought (CoT) ⭐
让模型逐步推理：

```
问：一个商店有 23 个苹果，卖了 15 个，又进货了 8 个，现在有多少个？

请一步步思考：
1. 原来有 23 个苹果
2. 卖了 15 个：23 - 15 = 8
3. 进货了 8 个：8 + 8 = 16
答：现在有 16 个苹果
```

#### Zero-Shot CoT
只需加一句 "请一步步思考" 即可触发 CoT

#### Auto-CoT
让模型自动生成推理链

#### Self-Consistency
多次采样，取多数投票的答案

## 2. 高级提示技巧

### 2.1 ReAct (Reasoning + Acting)
交替推理和行动：
```
问题：2023年诺贝尔物理学奖获得者是谁？

思考1：我需要搜索2023年诺贝尔物理学奖信息
行动1：Search[2023年诺贝尔物理学奖]
观察1：2023年诺贝尔物理学奖授予...
思考2：我已经得到了答案
回答：...
```

### 2.2 Tree-of-Thought (ToT)
- 将推理过程组织为树结构
- 每步探索多个分支
- 回溯选择最优路径

### 2.3 Graph-of-Thought (GoT)
- 推理过程建模为图
- 允许合并和分支
- 更灵活的推理结构

### 2.4 Self-Refine
模型自我迭代优化：
```
1. 生成初始回答
2. 自我批评：指出不足
3. 根据批评修改
4. 重复 2-3 直到满意
```

### 2.5 Directional Stimulus Prompting
用提示词引导模型关注特定方向：
```
请从技术创新角度分析以下公司...
```

## 3. 系统提示设计

### 3.1 系统提示最佳实践
```
你是一个专业的[角色]。

## 职责
- [职责1]
- [职责2]

## 规则
- [规则1]
- [规则2]

## 输出格式
[格式要求]

## 示例
[示例]
```

### 3.2 常见模式

| 模式 | 描述 | 示例 |
|------|------|------|
| 角色扮演 | 指定专业角色 | "你是一个资深的Python开发者" |
| 输出约束 | 限制输出格式 | "请以JSON格式输出" |
| 分步引导 | 指定思考步骤 | "第一步...第二步..." |
| 示例驱动 | 提供输入输出示例 | "以下是几个示例..." |
| 负面约束 | 说明不要做什么 | "不要使用专业术语" |

## 4. 提示攻击与防御

### 4.1 攻击类型
| 攻击 | 描述 |
|------|------|
| Prompt Injection | 注入恶意指令覆盖原始提示 |
| Jailbreak | 绕过安全限制 |
| Prompt Leaking | 诱导泄露系统提示 |
| Data Extraction | 提取训练数据 |

### 4.2 防御策略
- 输入/输出分隔符
- 指令层级化
- 输入验证与清洗
- 输出过滤
- 红队测试
