## 2. 数据清洗 Pipeline

### 2.1 预处理
```
原始 HTML → 文本提取 → 语言检测 → 去重 → 质量过滤 → 有害内容过滤 → 最终数据
```

### 2.2 关键清洗步骤

#### 文本提取
- Trafilatura / BeautifulSoup 提取正文
- 去除导航栏、广告、模板文本

#### 语言检测
- fastText 语言检测
- 保留目标语言，过滤噪声语言

#### 去重 ⭐
| 方法 | 粒度 | 效果 | 速度 |
|------|------|------|------|
| Exact Dedup | 文档级 | 去完全重复 | 快 |
| MinHash LSH | 文档级 | 去近似重复 | 中 |
| Suffix Array | 段落级 | 去子文档重复 | 慢 |
| SemDeDup | 语义级 | 去语义重复 | 慢 |

#### 质量过滤
- **Perplexity 过滤**：用 KenLM 计算困惑度，过滤异常文本
- **分类器过滤**：训练质量分类器，过滤低质量文本
- **启发式规则**：
  - 最少词数 / 最多词数
  - 平均词长范围
  - 特殊符号比例
  - 重复行比例

#### 有害内容过滤
- PII (个人隐私信息) 检测与脱敏
- 仇恨言论/暴力内容过滤
- 色情内容过滤

## 3. 分词与数据准备

### 3.1 Tokenization
- 统一分词器训练
- 多语言特殊处理
- 预分词 (Pre-tokenization) 规则

### 3.2 数据格式
```json
{"text": "document content here...", "meta": {"source": "wikipedia", "lang": "zh"}}
```

### 3.3 数据混合 (Data Mixing)
- 不同数据源按权重采样
- 温度参数控制采样分布：
$$P_i = \frac{w_i^{1/T}}{\sum_j w_j^{1/T}}$$

## 4. 合成数据

### 4.1 为什么需要合成数据
- 高质量人类数据逐渐耗尽
- 特定能力（推理、代码）数据稀缺

### 4.2 合成数据生成方法
- **Self-Instruct**：用强模型生成指令-回答对
- **Evol-Instruct**：逐步演化指令复杂度
- **Magpie**：利用聊天模板自动生成
- **代码执行验证**：生成代码+执行验证

### 4.3 合成数据风险
- 模型崩溃 (Model Collapse)：递归训练导致退化
- 分布偏移：合成数据分布窄于真实数据
- 解决：混合真实数据 + 质量控制
